A) Introduction - top

Nova Gallery WP makes it easy to embed the Nova Gallery – Responsive HTML5 Multimedia Gallery, available at Codecanyon, in your Wordpress theme. It displays all your images, audio files and video files in a sleek and attractive interface. You have the option of presenting your items in two different modes, which are a Thumbnail Grid and Fullwidth view, and you also have the option of switching between the two modes. You can have multiple sets/albums of items and can filter the gallery items based on file type or based on custom categories. The gallery features a fluid responsive design and can fit in any screen size ranging from mobile phones to desktop browsers. The gallery also features touch-screen support and uses hardware accelerated CSS based animations wherever possible which results in smooth animations, that is especially noticeable in mobile devices. Nova Gallery WP allows you to create multiple galleries where each gallery can have as many items/sets as you want. Although Nova Gallery WP allows you to create multiple galleries you can only embed one gallery per post/page.

Please note that the CSS3 based animations and effects and other advanced HTML5 features will not work in older browsers such as IE9 and below. For older browsers suitable fallbacks have been provided so that the user experience and aesthetics of the gallery are not reduced. Hence visitors to your web page using old browsers will still see a nicely working gallery.

B) Installing the plugin - top

To install the plugin follow the steps below:

  1. Log in to the Wordpress admin panel.
  2. Head over to Plugins section and click on the “Add New” button
  3. Then click “Upload” to bring up the plugin upload dialog. Select the plugin file and upload it. Alternatively you could also unpack the plugin zip file in your computer and then upload it to /wp-content/plugins via FTP.
  4. Next go to the Plugins main area and you will see Nova Gallery WP being listed as one of the available plugins.
  5. Click the “Activate” button next to it and you are done.

If you are upgrading from an older version then just replace the old plugin files in your server with the new ones. All your previously created galleries will be retained when upgrading.

C) How to Use - top

After activating the plugin you will see that a new menu item has been created on the left sidebar called Nova Gallery. This menu item has two sub menus - Gallery Manager and Create New Gallery. These names are self explanatory and whenever you want to create a new gallery you will head over to the Create New Gallery page, and all your existing galleries will be listed in the Gallery Manager page, from where you can edit or delete galleries, and also get the shortcodes for embedding galleries.

So first of all to create a new gallery head over to the appropriate page and you will be presented with a list of options. They are described below:

Gallery name

You must provide a name to the gallery that you are going to create. This is a required field and you will receive an error if you leave it empty. Although different galleries can have the same name but it is advisable that you provide unique names so that it is helpful for you to quickly differentiate between galleries.

Gallery description

You can write a description for the gallery here. This field is optional.

Gallery Options

A list of options that are applicable to the gallery. Their functions are described as follows:

  • Colour Scheme - Two colour schemes are available. Choose one from the available list.
  • Starting mode - Choose whether the Thumbnail Grid or the Fullwidth mode will be shown when the page first loads.
  • Resize to fit media in Fullwidth mode - By default the images/video loaded in the Fullwidth mode, when the page first loads, are shown in their actual size, even if that exceeds the gallery container. But if this option is enabled then the images/video in Fullwidth mode, when the page first loads, will be resized maintaining aspect ratio to fit inside the gallery container.
  • Show Thumbnails in Fullwidth mode - You can enable this option to show the item thumbnails in Fullwidth mode when the gallery first loads.
  • Appearing animation for Thumbnail Grid - Choose the animation that will be used to make the thumbnails appear on page, when the page first loads in the Thumbnail Grid mode.
  • Animation for captions in Thumbnail Grid - You can choose the animation for showing captions for items in Thumbnail Grid mode.
  • Always show item titles - You can choose to make the gallery item titles in Thumbnail Grid mode always visible, instead of only showing them on hover.
  • Spacing between thumbnails - If you want some spacing between the item thumbnails in Thumbnail Grid mode then you can mention that value here in px. By default this is zero.
  • Transition effect in Fullwidth mode - Choose the transition effect for switching between items in Fullwidth mode.
  • Animation for Gallery Sets page - If you have created sets for your gallery then you can choose the animation for showing/hiding the Gallery Sets page.
  • Display type of the gallery - If the gallery needs to stretch across the browser window and is a body level element, then choose “window”, else if the gallery should only stretch across its parent container then choose “container”.
  • Open Links in New Windows - You can choose to open the links associated with gallery thumbnails in a new window/tab instead of the same window/tab as the gallery (which is the default option).
  • Youtube Thumbnails - If this is enabled then thumbnails for Youtube videos will be pulled from Youtube.
  • Vimeo Thumbnails - If this is enabled then thumbnails for Vimeo videos will be pulled from Vimeo.
  • Shuffle Gallery items - If this is enabled then gallery items will be randomly ordered when the page refreshes every time.
  • Preload Images - You can enter the number of items whose full-size images will be preloaded in the background for the Fullwidth mode. This allows for smoother viewing of items in this mode. But please don't enter a large number as that can result in unnecessary downloading of large images which the user may never view, and is most definitely not recommended for mobile devices since they have lower bandwidth.
  • Fullwidth Slideshow - If you want to start the slidehsow in Fullwidth mode when the page first loads then enable this option.
  • Pause interval for slideshow - You can set the time interval, in ms, for which an item will remain visible during the slideshow before advancing to the next item.
  • Stop slideshow when page hidden - By default the gallery pauses the Fullwidth mode slideshow (inlcuding any audio/video playback) when the page loses focus. This helps prevent unnecessary consumption of resources, which is especially desirable for mobile devices. But if for whatever reason you want to disable this feature then you can do so.
  • Detect Mobile Devices - You have the option of detecting mobile devices and serving them low resolution videos, which you can upload below.
  • Autoplay Video and Audio - Enable this to start playing video/audio files automatically when the media player shows up. This setting will also affect Youtube/Vimeo videos.
  • Play in a Loop - Enable this option to play video/audio files continuously in a loop. This setting will not affect Youtube/Vimeo videos.
  • Store Volume Level - Enable this option to retain the volume level of the player when it is closed and volume will be set to that level when the player is opened again. This setting will not affect Youtube/Vimeo videos.
  • Custom Categories - Enter a comma separated list of custom category names that will be displayed in the filter menu bar at the top in the gallery.
  • Show File Type Buttons - Enable this option to show file type buttons (photo, audio, video) in the filter menu. If you have custom categories then you may not need to show these buttons.
  • Hide Gallery Menu - You can choose to hide the gallery menu completely.
  • Starting Category - Choose the starting category whose items will be shown when the page first loads.
  • Hide Thumbnail Grid Mode - If you don't want the Thumbnail Grid mode then you can enable this. Please note that for this to take effect you will also have to set the starting mode to Fullwidth. When this is enabled then the View mode buttons in the gallery menu bar will also be hidden.
  • Hide Fullwidth Mode - If you don't want the Fullwidth mode then you can enable this. Please note that for this to take effect you will also have to set the starting mode to Thumbnail Grid. When this is enabled then the View mode buttons in the gallery menu bar will also be hidden.
  • Flickr/Picasa Cache - Enable this option to cache Flickr/Picasa API query results.
  • Time interval for caching - You can set the time interval, in minutes, for which the the Flickr/Picasa API data will be cached before making a fresh call again.
  • Flickr Photos - If you want the gallery show photos from Flickr then enable this option. When this is enabled then you will be presented with options to configure the pulling of photos from Flickr.
  • Picasa Photos - If you want the gallery to show photos from Picasa then enable this option. When this is enabled then you will be presented with options to configure the pulling of photos from Picasa.

Flickr Options

Here you are given the options to configure the pulling of photos from Flickr and choose the manner of displaying them.

  • API Key - Your Flickr API key is required in order to pull photos from Flickr.
  • Flickr Source - You can choose the source from which photos are pulled. Available options are “text search”, “tags”, “user”, “set”, “group” and “collection”.
  • ID of Source - If you choose the source as “user” or “set” or “group” or “collection” then you need to mention the ID number of that. If your source is “text search” then write down the search string. If your source is “tags” then write down a comma separated list of tag names.
  • User ID for Collection - If your source type is “collection” then you need to mention the user id of the owner of the collection.
  • Number of albums to show - If your sourceType is “collection” then you can choose to show only a specific number of albums from that collection, and that number (a positive integer) is passed to this parameter. By default all the albums in a collection is shown, which can be done by passing the string "all" to this parameter or leaving the field blank.
  • Number of Flickr photos - You can set the number of photos that will be pulled from Flickr. Your Flickr API key may have some restrictions on the number of photos that you can pull at a time.
  • Sort Order - You can choose how the photos from Flickr will be sorted and displayed in the page.
  • Thumbnail Size - Choose the size of the thumbnail that will be shown in the Thumbnail Grid mode.
  • Large Image Size - Choose the size of the large image that will be shown in the Lightbox in Thumbnail Grid mode and also in the Fullwidth mode.

Picasa Options

If you have chosen to show photos from Picasa then you can configure here how photos are pulled from Picasa and displayed in the page.

  • Source - Choose the source from which photos will be pulled from Picasa.
  • User Name - If your source is “user” then mention the username here. If your source is “album” or “collection” then you need to mention the username of the owner of that album or collection.
  • Album ID - If your source is “album” then you need to mention the id of that album here.
  • Search Text - If your source is “text search” then enter the search string here.
  • Number of albums to show - If your sourceType is “collection” then you can choose to show only a specific number of albums from that collection, and that number (a positive integer) is passed to this parameter. By default all the albums in a collection is shown, which can be done by passing the string "all" to this parameter or leaving the field blank.
  • Number of photos - You can enter the number of photos that will be pulled from Picasa.
  • Thumbnail Size - Choose the size of the thumbnail that will be shown in Thumbnail Grid mode.
  • Large Image Size - Choose the size of the large image that will shown in the Lightbox in Thumbnail Grid mode and also in Fullwidth mode.

Upload Gallery Files

In this section you are given the option to upload the gallery media files. There are three sections for separate uploading of image, audio and video files. Note that Nova Gallery uses the Wordpress Media Library so if any of your intended files are already present in the Library then you do not need to upload it again. Whatever file you upload in the Media Library you can select those files when configuring the gallery items below. Also note that if you intend to use the same image as both the thumbnail version and as the large version then uploading it only once will suffice and the gallery will generate the thumbnail for you based on the size you mention (more on that later).

Gallery Sets & Items

This section allows you to add gallery items and sets and also configure them. First of all you can choose whether you want to have sets/albums in the gallery. Based on that choice you will be presented with two different ways of adding items in the gallery. The items/sets that you create can be sorted into any order by dragging them, and that also holds for items present inside a set. You are also given the option to choose whether the thumbnails that are generated by the plugin are cropped from the original image based on your specified size or will they be scaled maintaining aspect ratio. The configuration options for the items are described below.

  • Set Name - If you have enabled sets in your gallery then when you create sets you can mention the name of the set here, that will displayed in the gallery sets page.
  • Set Thumbnail - You can choose the thumbnail for representing the set. This is optional, and if no thumbnail is set then the thumbnail of the first item in the set will be used.
  • File Type - Choose the file type of the item, i.e. whether it is a photo, audio or video.
  • Category - Enter a comma separated list of category names to which this item belongs. The list of category names entered here should be present in the list of custom categories entered earlier.
  • Thumbnail - Choose the image that will used as thumbnail for the item.
  • Thumbnail Size - If you have set the same image file as both the thumbnail and as the large image then enter the dimensions of the thumbnail here that will be used to generate the thumbnail from the chosen image file.
  • Large Image - Select the image that will be shown as the large image in the gallery for “photo” file type. If the file type is “audio” or “video” then this image will be used as the poster.
  • Audio Files - If your file type is audio then you will be given the option to select the mp3 and ogg formats of your audio file. If you only upload the mp3 format then in browsers that don't support it natively, like Firefox, Flash will be used to play that file.
  • Video Files - If your file type is video then you will be given the option to select the mp4, webm and ogv formats of your video file. If you chose to detect mobile devices then you are given the option to upload a low resolution mp4 video file specifically for mobile devices. If you only upload a mp4 file then Flash will be used to play it in browsers that don't support it natively.
  • Youtube Video - If you chose video file type then you can enter a Youtube video link instead of uploading files. The link should be of the form http://www.youtube.com/watch?v=abcdefghijk. Any other form of the link won't work.
  • Vimeo Video - If you chose video file type then you can enter a Vimeo video link instead of uploading files. The link should be of the form http://vimeo.com/12345678. Any other form of the link won't work.
  • Title - Enter the title for the gallery item, which will be displayed in the Coverflow and Thumbnail modes and also in the Lightbox.
  • Caption - Here you can write a short description of the gallery item. This field can contain any html code.
  • Link - If you would like the thumbnails of the gallery items in Thumbnail Grid mode to act as links to external pages then you can enter the url of that link here. In Fullwidth mode the large image will act as the link.

Whenever you want to create a new gallery then the form on that page will be filled out with default option values, so that you only have to create the gallery items in order to make your gallery work. You can just change the settings that you want to customise and save the gallery settings. Needless to say, whenever you are going to edit an existing gallery, the same page will open and all the form fields will be filled out with the settings with which you created the gallery.

If you want to delete an uploaded file then you can delete it directly from the WP Media Library or you can open the uploading modal box from the uploading section in the gallery and find the file from the library and delete it. Just clear out the text field for that file in the item settings and save your settings.

Multimedia files

As already stated Nova Gallery supports images, audio and video. When viewing images in the Lightbox in Thumbnail Grid mode, if the dimensions of the image is bigger than the browser window dimensions then that image will be resized to fit inside the browser window. For playing audio and video Nova Gallery uses Mediaelement.js. This is a HTML5 audio and video player. For browsers that do not support HTML5 audio or video playback, Mediaelement.js enables fallback to Flash and Silverlight. Now different browsers require different file formats for native playback of HTML5 audio and video. For audio you must upload a mp3 version of the file which is supported by Safari, Chrome and IE9. You must upload an ogg version if you want to have native playback in Firefox and Opera. The ogg format is also supported by Chrome. But you can upload only the mp3 version and Mediaelement.js will use Flash to play the audio file in Firefox and Opera. Similarly for the case of video playback, if you want to have a wide range of browsers natively play the video, then you must upload mp4, webm and ogv versions of the video. But here also you can only upload a mp4 version (h.264) and Mediaelement.js will use Flash or Silverlight to play the video in browsers that do not support that format. If you use only mp4 (h.264) videos and find that when there is Flash fallback in unsupported browsers, the entire video needs to be downloaded before it can be watched, then the index of the mp4 file is placed at the end. To remedy this case use the utility QTIndexSwapper on your mp4 files. If you find that after uploading Nova Gallery to your server your browser is not able to play video files then you may need to configure your server in order to enable it to stream video files. For an Apache server place the following code in an .htaccess file and place it at the root of your website. 

AddType video/ogg .ogv
AddType video/mp4 .mp4
AddType video/webm .webm

Keyboard Shortcuts

Nova Gallery also provides some handy keyboard shortcuts to facilitate easier browsing. You can press LEFT and RIGHT keys to flip through the items in the Fullwidth mode and also when the Lightbox is open in Thumbnail Grid mode. In order to close the Lightbox in Thumbnail Grid mode you can press ESC. Pressing ESC will also close the caption overlay in Fullwidth mode.

Responsive Design

Nova Gallery features a fluid responsive design and thus can adapt itself to be viewed in desktop browsers as well as in mobile devices. You can also place the gallery in any container of a particular width (choose display style as “Container” for this case). By default the gallery will stretch across the entire browser window. To have the best effect for the responsive design the gallery should be in a page all by itself and as a body (root) level element, meaning that it should not be placed inside any container. That way the responsive design of the gallery will function fully when the browser width changes. You can see this effect by opening the demo of the gallery in Codecanyon in your browser and resizing its width.

D) Embed galleries - top

All the galleries that you create will be visible in the Gallery Manager page. When you go to that page you will see that each gallery has a field in the table titled shortcode. In this field you will see the shortcode that is required for embedding a particular gallery. Just copy the shortcode for a gallery, that is displayed there and then head over to Posts section in Wordpress. Open any post or page where you want to have a gallery, in the Wordpress editor, and then paste the shortcode at a place where you find suitable, and then save or publish that post or page. Then open that post/page in your browser and you will see the gallery in action with all the settings that you created it with. If you want to directly write the embed code in a Wordpress template file then you will have to use the do_shortcode function. Using this function has the advantage that you can embed a gallery in a part of the Wordpress template where content is not pulled using the Wordpress Loop. An example usage of this function to embed a gallery is: 

    <?php echo do_shortcode('[novagallery id="1"]'); ?>

Special Notes for IE

IE8 and below needs a special javascript file to be declared in the header of your WP theme, in the <head>, in order to make it recognize new HTML5 tags that are required to make the entire gallery work correctly. You should declare the file in the header in the following way: 

<!--[if lt IE 9]>
	<script src="http://html5shiv.googlecode.com/svn/trunk/html5.js"></script>
<![endif]-->

IE also needs the CSS3 PIE file in order for modern CSS3 styles that this gallery uses, to be rendered there. You must place the PIE.htc file that is present in the css folder of the plugin, in the root of your site so that it is loaded in IE. Also IE9 is very strict regarding mime-types, so if you find that the htc file is not being loaded in IE9 then you must set the correct mime-type as 

AddType text/x-component .htc
